### 1.2 Datenbank-Anbindung
- **Live-Daten:** `https://timoboese.com/pano/berlin/data.json`
- **Format:** `[filename, name, district, lat, lng, tags, arucoId]`
- **38 Einträge** mit ArUco-IDs 1-38
- **Fallback:** 39 Locations in `panoramaLocations.ts` (IDs 1-39)
- **Fetch:** `fetchLocationsFromDB()` beim App-Start → überschreibt Fallback
- **Lookup:** `findLocationById(id)` → sucht in Live-Daten, sonst Fallback

### 1.3 ArUco-Scanner
- **Bibliothek:** `js-aruco2` (pure JS, kein Native Code)
- **Dictionary:** ARUCO (7x7 Marker, 1024 IDs)
- **Pipeline:** Foto → skalieren (640px) → Base64 → JPEG decodieren → Graustufen → Marker erkennen
- **IDs 1-38** sind in der Datenbank zugewiesen
- **User-Strings:** "ArUco" wurde aus allen sichtbaren Texten entfernt → "Code"

## 2. SCREENS & ABLAUF

### 2.1 Intro (`screen === 'intro'`)
- **Video:** `assets/intro.mp4` (expo-video)
- **Interaktion:** Tap auf "Überspringen" → Tutorial
- **Automatisch:** Video endet → Loading

### 2.2 Loading (`screen === 'loading'`)
- **Dauer:** 2,5 Sekunden
- **Quote:** Zufälliger Reise-Spruch
- **DB-Fetch:** `fetchLocationsFromDB()` wird hier gestartet
- **Navigation:** Automatisch → Tutorial

### 2.3 Tutorial (`screen === 'tutorial'`)
- **4 Slides** mit Swipe + Klick-Navigation:
  1. "Eine Aufgabe. Nur eine." (dunkel, rot)
  2. "Ziehen. Scannen. Die Zeit läuft." (orange, dunkel)
  3. "Wo zur Hölle bist du?" (dunkel, orange)
  4. "Auf die harte Tour?" (rot, dunkel)
- **Navigation:**
  - **Swipe:** Horizontal mit `pagingEnabled`
  - **Klick linke Hälfte:** Vorheriger Slide
  - **Klick rechte Hälfte:** Nächster Slide (letzter → Setup)
  - **Swipe über letzten Slide:** → Setup
- **Dots:** Mittig unter den Slides

### 2.4 Setup (`screen === 'setup'`)
- **Logo:** `GeoChecker_Aruco_Square_01.png`
- **Spieler:** 2+ (default 2, max unbegrenzt)
  - Name eingeben (TextInput)
  - Karte zuweisen (Button → City Scanner)
  - Entfernen (✕ Button, nur ab 3 Spielern)
- **Timer:** Slider (1, 5, 10, 20, 30, 45 Sekunden)
- **Runden:** 5, 10, 15 pro Spieler
- **Start:** Nur wenn alle Spieler eine Karte haben

### 2.5 City Scanner / QR Scanner
- **Kamera:** `expo-camera` (CameraView)
- **Zwei Modi:**
  1. **City Scanner** (Setup): Karte einem Spieler zuweisen
  2. **QR Scanner** (Game): Karte ziehen für aktuelle Runde
- **Erkennung:**
  - **Barcode:** QR, Code128, Code39, EAN13, EAN8
  - **OCR:** `@react-native-ml-kit/text-recognition` (Foto → Text)
  - **Manuell:** Code (#042) oder Stadtname eingeben
- **OCR-Pipeline:** Foto → `TextRecognition.recognize()` → über `result.blocks` iterieren → erste Zahl/Ortsname nehmen
- **Debug-Logs:** `[OCR]` Prefix in console.log

### 2.6 Game (`screen === 'game'`)
#### Phase: scan-qr
- Runden-Anzeige, Spielername, Punktestand
- "KARTE ZIEHEN & SCANNEN" → QR Scanner
- "NEUE KARTE LADEN" → überspringt Scan

#### Phase: view
- **Panorama:** 360°-Bild von `timoboese.com/pano/berlin/{id}.avif`
- **Timer:** Countdown (oben mittig)
  - Letzte 10 Sekunden: Puls-Animation + Vibration
  - Bei 0: Automatisch → Pick-Phase
- **Loading:** "Panorama wird geladen..." + "NEUE KARTE LADEN" nach 5s
- **Error:** "Fehler beim Laden" + "NEUE KARTE LADEN"
- **"ORT WÄHLEN"** Button (unten)

#### Phase: pick
- Liste aller Städte auf dem Tisch
- Spieler wählt den Ort, der am nächsten liegt
- Distanz wird berechnet (Haversine)

#### Phase: challenge
- Anzeige: Gewählter Ort + Distanz
- Anzeige: Nächster Ort + Distanz
- **Challenge:** Andere Spieler können Token einsetzen (falsch = Punkt verloren)
- "AUFLÖSEN" → Ergebnis

#### Phase: result
- **Richtig:** ✓ + Punkt für aktiven Spieler (oder Challenger)
- **Falsch:** ✗ + kein Punkt
- **Tie-Breaker:** Bei Gleichstand am Ende → Extra-Runde
- "NÄCHSTE RUNDE" / "ERGEBNIS ANZEIGEN"

### 2.7 Result (`screen === 'result'`)
- Rangliste (sortiert nach Punkten)
- Gewinner-Anzeige
- "NOCH EIN SPIEL" → zurück zu Setup
- "Zum Startbildschirm" → zurück zu Intro

## 6. BEKANNTE BUGS & OFFENE PUNKTE

### 6.1 OCR funktioniert nicht zuverlässig
- **Problem:** `@react-native-ml-kit/text-recognition` erkennt oft keinen Text
- **Debug:** `console.log('[OCR] ...')` eingebaut – Logs zeigen ob Text ankommt
- **Workaround:** Manuelle Eingabe (#042 oder Stadtname)
- **Nächster Schritt:** ML Kit Modell prüfen, ggf. auf Google ML Kit Vision umstellen

### 6.2 ArUco-Scanner (useArucoScanner) wird nicht aktiv genutzt
- **Problem:** Der ArUco-Scanner ist eingebaut, aber der `scanCard`-Aufruf fehlt in der UI
- **Aktuell:** Der Scanner läuft im Hintergrund, aber es gibt keinen Button der `scanCard()` triggert
- **Workaround:** Barcode-Scanner + OCR + manuelle Eingabe
- **Nächster Schritt:** ArUco-Scan-Button in der Scanner-UI einbauen

### 6.3 Fallback-Locations sind veraltet
- **Problem:** `FALLBACK_LOCATIONS` in `panoramaLocations.ts` hat andere Namen/IDs als die DB
- **Beispiel:** ID 20 = "Bellevue Palace" (Fallback) vs "Gaudystraße 9" (DB)
- **Lösung:** `fetchLocationsFromDB()` überschreibt beim Start – funktioniert nur mit Internet

### 6.4 Kein Offline-Modus
- **Problem:** Panorama-Bilder, DB-Fetch und ArUco-Codes brauchen Internet
- **Nächster Schritt:** Caching für Panoramen und DB-Daten

### 6.5 Build-Dauer
- **Problem:** GitHub Actions Build dauert ~15-20 Minuten
- **Workaround:** `npx eas build --platform android --profile preview` für lokale Builds
